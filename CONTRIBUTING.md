# Contributing to DevMohammad-SA's Dotfiles

Thank you for your interest in contributing! While these are personal dotfiles, suggestions and improvements are always welcome.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists in the [Issues](https://github.com/DevMohammad-SA/dotfiles/issues) section
2. If not, create a new issue with:
   - A clear, descriptive title
   - Detailed description of the issue or suggestion
   - Steps to reproduce (for bugs)
   - Your environment details (OS, shell version, etc.)

### Suggesting Enhancements

Feel free to suggest:
- New configurations or tools
- Theme improvements
- Better organization
- Additional documentation
- Installation script improvements

### Pull Requests

If you'd like to contribute code:

1. **Fork** the repository
2. **Create a branch** for your feature:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**:
   - Keep changes minimal and focused
   - Test your changes thoroughly
   - Follow the existing code style
   - Update documentation if needed
4. **Commit** your changes:
   ```bash
   git commit -m "Add: brief description of changes"
   ```
5. **Push** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Submit a Pull Request** with:
   - Clear description of changes
   - Why the changes are beneficial
   - Any testing you've performed

## Guidelines

### Code Style

- Use consistent formatting with existing files
- Add comments for complex configurations
- Keep configurations modular and reusable
- Avoid hard-coded paths where possible

### Documentation

- Update README.md if you change structure or add features
- Add comments to configuration files for clarity
- Create or update individual README files in directories

### Testing

Before submitting:
- Test your changes in a clean environment
- Ensure installation script works (if modified)
- Verify symbolic links are created correctly
- Check that configurations don't break existing setups

## Questions?

If you have questions about contributing, feel free to:
- Open an issue for discussion
- Comment on existing issues or PRs

## Code of Conduct

Be respectful and constructive in all interactions. This is a personal project, but all contributors are welcome and appreciated!
